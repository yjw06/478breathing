# 378 호흡 (4-7-8 Breathing) — 설계 문서

작성일: 2026-06-25

## 목적

밤에 잠들기 어려울 때 4-7-8 호흡법을 따라 할 수 있도록 돕는 웹앱.
아이패드에서 잠들기 전 사용한다. 눈을 감고 귀로 따라가는 경험이 핵심이며,
화면은 어둡고 최소한으로 유지한다.

## 사용 환경

- **기기**: 아이패드
- **실행 방식**: 로컬 HTML 파일. 파일 앱에 넣어두고 Safari로 연다.
- **네트워크**: 불필요 (완전 오프라인 동작)
- **서버**: 없음

## 핵심 경험 (단일 화면, 단일 흐름)

1. 파일을 열면 어두운 화면에 작은 **설정 영역**과 큰 **"시작" 버튼**.
2. 시작을 누르면:
   - 짧은 준비 멘트
   - **들숨 → 멈춤 → 날숨**을 설정한 횟수만큼 음성으로 안내
   - 끝나면 마무리 멘트("다 됐어요, 편히 주무세요" 류)
3. 끝나면 다시 정지(시작) 화면으로. 다시 누르면 반복.

## 설정 (사용자 조절 가능)

시작 화면에 노출되는 항목:

| 항목 | 기본값 | 비고 |
|------|--------|------|
| 들숨 (초) | 4 | |
| 멈춤 (초) | 7 | |
| 날숨 (초) | 8 | |
| 반복 횟수 | 4 | |

- 설정값은 `localStorage`에 저장하여 다음 실행 시 마지막 값 유지.
- 입력 UI는 단순하게(숫자 스텝퍼 또는 +/- 버튼). 야간 사용이므로 큰 터치 타깃.

## 소리 설계 (외부 오디오 파일 0개)

- **음성 안내**: 브라우저 내장 음성합성(`SpeechSynthesis`, `ko-KR`)으로
  각 단계 시작 시 "들이쉬세요 / 멈추세요 / 내쉬세요" 안내.
  - 사용자 제스처(시작 버튼) 이후 발화하므로 iOS 자동재생 정책 충족.
  - `getVoices()`가 초기에 비어 있을 수 있으므로 `voiceschanged` 이벤트 대기 후 한국어 음성 선택. 한국어 음성이 없으면 기본 음성으로 폴백.
- **배경음**: `Web Audio API`로 생성한 은은한 노이즈(부드러운 백색/브라운 노이즈)를
  세션 동안 낮은 볼륨으로 재생. 외부 파일 없이 코드로 합성.
- **전환음(차임)**: 각 단계가 바뀔 때 아주 부드러운 차임 한 번. Web Audio로 합성.

## 화면 설계 (시각 애니메이션 없음, 최소 표시)

- 거의 검은 배경(눈부심 최소화).
- 세션 중 현재 단계 글자("들이쉬기" 등)와 남은 사이클 표시(예: ●●○○)를 **흐릿하게**만.
- 눈을 감아도 되고 떠도 부담 없는 밝기.

## 기술 구조

- **단일 `index.html`** — HTML / CSS / JS 전부 인라인. 외부 의존성·인터넷·서버 불필요.
- **상태 기계**로 단계 진행: `대기 → 준비 → 들숨 → 멈춤 → 날숨 → (반복) → 마무리 → 대기`.
- 타이밍은 단계 종료 시각(timestamp) 기준으로 계산하여 누적 오차 방지
  (`setTimeout` 단독 사용 시의 드리프트 회피).
- 음성/차임/배경음을 단계 전환 훅에서 트리거.

### 모듈 경계 (단일 파일 내 논리 분리)

- **타이머/상태 기계**: 단계 순서와 시간 관리. 입력은 설정값, 출력은 "현재 단계 변경" 이벤트.
- **오디오**: 음성 발화, 배경음 on/off, 차임 재생. 상태 기계 이벤트를 구독.
- **화면**: 현재 단계·남은 사이클 표시. 상태 기계 이벤트를 구독.
- **설정 저장/로드**: `localStorage` 읽기/쓰기.

각 부분은 "현재 단계 변경" 이벤트라는 단일 인터페이스로 통신한다.

## 알아둘 제약

- **화면 자동 잠금 시 소리 멈춤**: 웹페이지 특성상 iOS에서 화면이 잠기면 오디오가 중단된다.
  - 기본 4-7-8 × 4회 ≈ 1분 20초로 보통 자동잠금(2분 이상) 전에 끝난다.
  - 단, 사용자가 횟수/시간을 늘리면 길어질 수 있으므로, 시작 화면에
    "세션 중 화면이 꺼지지 않게 자동잠금을 길게 설정하세요" 안내 문구를 둔다.
  - (선택적 개선) iOS 16.4+ Wake Lock API로 화면 켜두기 시도 — 지원 시에만.

## 테스트 관점

- 타이밍 정확도: 설정한 초 합계와 실제 단계 진행 시간 비교(드리프트 허용 오차 내).
- 설정 저장/로드: 값 변경 후 재로드 시 유지되는지.
- 오디오 시작: 시작 버튼(사용자 제스처) 이후에만 발화/재생 시작되는지.
- 폴백: 한국어 음성 없을 때 기본 음성으로 동작하는지.
